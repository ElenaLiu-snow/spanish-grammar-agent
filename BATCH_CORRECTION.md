# 批改完整性优化说明

## 问题

用户反馈批改内容不完全，可能只批改了部分练习。

## 解决方案

### 1. 增加 Token 限制

**之前**: max_tokens = 8192
**现在**: max_tokens = 16384

这样可以支持更长的批改输出。

### 2. 优化系统提示词

在 `backend/prompts/system.txt` 中添加了批改练习的具体要求：

- **完整性要求**: 必须批改所有题目，不要遗漏
- **进度标注**: 如果内容过长，标注"（第X题共Y题）"
- **标准格式**: 每个题目按固定格式批改

### 3. 批改格式模板

```markdown
## 第X题：[题目名称]

### 学生答案
[学生的答案]

### 正确答案
[正确的变位]

### 错误分析
- 具体错误
- 语法规则
- RAE引用

### 语法讲解
[详细讲解]
```

## 使用建议

### 如果批改仍然不完整

1. **分批批改**: 如果文档很长，可以分几次上传
   - 第一次：批改第1-10题
   - 第二次：批改第11-20题

2. **明确要求**: 在问题中明确说明"请批改所有15道题目"

3. **检查输出**: 如果发现批改中断，可以追问：
   - "继续批改剩余的题目"
   - "第10题以后的批改呢？"

### 示例提问

**推荐**:
```
请批改这份练习的所有题目，共15道题。每道题都要：
1. 指出正确答案
2. 标出错误
3. 解释语法规则
```

**如果中断**:
```
继续批改第8题到第15题
```

## 技术说明

- 最大输出长度: 16384 tokens（约8000-10000中文字符）
- 流式响应: 实时显示批改内容
- 完整性保证: 系统提示词强调不遗漏任何题目
